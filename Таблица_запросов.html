Таблица запросов для тестирования API

Базовый URL: https://cvb-363x.onrender.com
№ Название теста Метод URL / Эндпоинт Параметры Body (JSON) Ожидаемый код Ожидаемый результат
1 Проверка работоспособности GET /health - - 200 {"status": "ok", "service": "electronics-store-api"}
2 Получить все товары GET /api/products - - 200 Массив товаров
3 Получить товары по категории GET /api/products category=Смартфоны - 200 Массив товаров категории "Смартфоны"
4 Получить товар по ID (существующий) GET /api/products/1 - - 200 Объект товара с id=1
5 Получить товар по ID (несуществующий) GET /api/products/999 - - 404 {"error": "Товар не найден"}
6 Получить все категории GET /api/categories - - 200 Массив строк категорий
7 Добавить товар (корректный) POST /api/products - { "name": "Клавиатура Logitech", "description": "Механическая клавиатура", "price": 3500, "category": "Аксессуары", "inStock": 20 } 201 {"success": true, "productId": X, "message": "Товар успешно добавлен"}
8 Добавить товар (без обязательных полей) POST /api/products - { "description": "Товар без названия" } 400 {"error": "Необходимо указать название и цену товара"}
9 Добавить товар (без цены) POST /api/products - { "name": "Товар без цены" } 400 {"error": "Необходимо указать название и цену товара"}
10 Получить все заказы GET /api/orders - - 200 Массив заказов
11 Получить заказ по ID (существующий) GET /api/orders/1 - - 200 Объект заказа с массивом items
12 Получить заказ по ID (несуществующий) GET /api/orders/999 - - 404 {"error": "Заказ не найден"}
13 Создать заказ (корректный) POST /api/orders - { "clientName": "Иван Иванов", "clientEmail": "ivan@example.com", "clientPhone": "+79001234567", "items": [ { "productId": 1, "quantity": 2 } ] } 201 {"success": true, "orderId": X, "totalAmount": Y, "message": "Заказ успешно создан"}
14 Создать заказ (без обязательных полей) POST /api/orders - { "clientEmail": "test@example.com" } 400 {"error": "Необходимо указать имя, email и товары"}
15 Создать заказ (пустой массив товаров) POST /api/orders - { "clientName": "Иван Иванов", "clientEmail": "ivan@example.com", "items": [] } 400 {"error": "Необходимо указать имя, email и товары"}
16 Создать заказ (несуществующий товар) POST /api/orders - { "clientName": "Иван Иванов", "clientEmail": "ivan@example.com", "items": [ { "productId": 999, "quantity": 1 } ] } 400 {"error": "Товар с id 999 не найден"}
17 Создать заказ (недостаточно товара) POST /api/orders - { "clientName": "Иван Иванов", "clientEmail": "ivan@example.com", "items": [ { "productId": 1, "quantity": 1000 } ] } 400 {"error": "Недостаточно товара с id 1 на складе. Доступно: X, запрошено: 1000"}
18 Создать заказ (несколько товаров) POST /api/orders - { "clientName": "Петр Петров", "clientEmail": "petr@example.com", "clientPhone": "+79007654321", "items": [ { "productId": 2, "quantity": 1 }, { "productId": 3, "quantity": 2 } ] } 201 {"success": true, "orderId": X, "totalAmount": Y, "message": "Заказ успешно создан"}
Примечания:
• Для POST-запросов необходимо добавить Header: Content-Type: application/json
• Для теста №3 используйте вкладку Params в Postman: Key = "category", Value = "Смартфоны"
• Для теста №11 сначала создайте заказ через тест №13, чтобы получить существующий ID
• X и Y в ожидаемых результатах - это числовые значения, которые будут возвращены сервером
